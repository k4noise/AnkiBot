## Задача 1: Базовая структура проекта
Приветственный экран - у вас есть <такие> колоды


<div>Термин - уникальная сущность (id) внутри одной колоды, создать вторую такую же нельзя</div>
<div>Все операции с карточками доступны только внутри колоды </div>
<div>Справка (перечень доступных команд): `/help`</div>
<div>Просмотр всех колод: `/list_decks`</div>

__Колода__:
- Создание колоды: `/create_deck <название колоды>`
- Переименование колоды: `/rename_deck <название колоды>:= <новое название>`
- Удаление колоды: `/delete_deck <название колоды>`, также приводит к удалению всех карт внутри колоды
- Просмотр карточек колоды: `/list_cards <название колоды> (термин = определение)`
 
__Карточка__:
- Создание карточки: `/create_card <название колоды>:<термин>=<определение>`
- Редактирование термина: `/edit_card_term <название колоды>:<термин>:=<новое название>`
- Редактирование определения: `/edit_card_def <название колоды>:<термин> =<новое определение>`
- Удаление карточки: `/delete_card <название колоды>:<термин>`
- Просмотр карточки: `/list_card <название колоды>:<термин>`

## Задача 2: Реализация режимов обучения
Обучение
Команда: `/check <режим обучения>`

__Режимы обучения:__

- С вводом терминов: `/check typing: <название колоды>`
Пользователю показывается определение и ожидается ввод термина. 

- С оценкой запоминания карточек: `/check memory: <название колоды>`
Пользователю показывается термин и предлагается выбор из трех вариантов: не помню, частично помню и помню

- С сопоставлением термина определению: `/check match: <название колоды>`
Пользователю показывается определение и термин случайным образом и предлагается указать, подходит ли термин определению. Если, по мнению пользователя, термин соответствует определению, то пользователь вводит 1, иначе - 0. 

<div>Обучение автоматически завершается, когда пользователю были показаны все карточки в колоде.<div>
<div>Возможен досрочный выход из режима обучения: `/end-check`.<div>

<div>Если колода пустая, то начать обучение невозможно и программа уведомляет пользователя об отсутствии карточек для изучения.<div>

## Задача 3: Реализация системы оценивания
Оценка карточек в системе обучения необходима для отслеживания прогресса пользователя и определения уровня усвоения термина. Система оценивания основывается на свойстве карточки “количество баллов”, которое лежит в интервале [0, 12]. В процессе обучения баллы изменяются: увеличиваются при правильных ответах и уменьшаются при неправильных.<br>
На оценку влияет количество верных ответов для конкретной карточки вне зависимости от количества повторений. Это означает, что для достижения 10 баллов потребуется минимум 5 повторений в режиме ввода термина и 10 в других режимах, при условии, что все ответы будут правильными. В случае ошибок количество необходимых повторений увеличивается.

Термин считается изученным, если имеет хотя бы 10 баллов; частично изученным, если имеет 5 баллов и не изученным в остальных случаях.

Режим обучения “ввод” имеет следующее распределение: неверный ввод => (-1)б, пропуск => (+0)б, верный ввод => (+2)б
Режим обучения “карточки” имеет следующее распределение: ”не помню” => (-1)б, “частично” => (0)б, “помню” => (+1)б
Режим обучения “соответствие” имеет следующее распределение: неверный ответ => (-1)б, пропуск => (+0)б, верный ответ=> (+1)б

Пользователь может видеть только статусы карточек (“изучены”, “частично изучены”, “не изучены”), но не сами баллы.

После выхода из режимов обучения будет показываться статистика текущего сеанса: для режима “карточки” – процент успешности изучения, для остальных режимов – соотношение правильных ответов к количеству показанных карточек.
Просмотр статистики доступен как для всех колод, так и для отдельной. Также статистика отображается при завершении обучения.

__Статистика по колоде (один сеанс)__:
Для режимов “ввод” и “сопоставление” указывается количество правильных ответов к общему количеству показанных карточек.
Для режима “карточки” указывается общий % успешности изучения показанных карточек по следующей формуле:<br>
Процент = 1/n * sum(50 + b_i * 50),<br>
где bi - балл, полученный пользователем за изучение отдельно взятой карточки (-1, 0 или 1), n - количество показанных карточек. Результат лежит в интервале [0, 100].

__Статистика по колоде (все сеансы)__: `/deck-stats <название колоды>` – показывает общее количество карточек и их статусы, т.е. отображает количество карточек согласно статусам “Изучено”, “Частично изучено” и “Не изучено”.

__Общая статистика (по всем колодам)__: `/stats` – показывает по каждой колоде среднюю оценку изученности карточек в процентах, которая вычисляется по формуле:<br>
Оценка колоды= 1/n * sum(b_i * 100 / 12),<br>
где bi - балл, полученный пользователем за изучение отдельно взятой карточки (-1, 0 или 1), n - количество показанных карточек. Результат лежит в интервале [0, 100].

__Пример статистики__:<br>
// общая статистика<br>
U: /stats<br>
P: Колода ‘Git’: 85%, колода ‘Maven’: 63%<br>
// статистика по колоде за все сеансы<br>
U:  /deck-stats Git<br>
P: Git: 2 карточки<br>
P: Изучено: 1<br>
P: В процессе: 1<br>
P: Осталось: 0<br>

где “Изучено” - количество карточек со статусом “Изученные” (10 баллов и более)<br>
“В процессе” - количество карточек со статусом “Частично изученные” ( от 5 до 9 баллов)<br>
“Осталось” - количество карточек со статусом “Не изучено” (до 4 баллов включительно)
